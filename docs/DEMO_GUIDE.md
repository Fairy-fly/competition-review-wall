# 演示流程

本文档用于课程设计答辩、简历项目录屏或 GitHub 项目展示。

## 演示前准备

1. 启动 MySQL。
2. 初始化数据库：

```bash
cd backend
npm run db:init
```

3. 启动后端：

在项目根目录执行：

```bash
cd backend
npm run dev
```

4. 启动前端：

另开一个终端，在项目根目录执行：

```bash
cd frontend
npm run dev
```

5. 打开：

```text
http://localhost:5173
```

## 推荐演示路线

### 1. 登录与首页测评墙

使用学生账号 `2023001` 登录。

重点讲解：

- JWT 登录后进入前端应用。
- 首页不是普通后台，而是“协作画像墙”。
- 用户卡片展示综合评分、再次组队率、技能方向、标签。
- 筛选区支持按专业、年级、标签、竞赛类型等查找队友。

### 2. 用户画像与雷达图

从首页点击任意队友画像，进入用户画像页。

重点讲解：

- 画像由匿名评价聚合生成。
- 综合评分、再次组队率、标签云体现协作风格。
- 雷达图展示任务、沟通、责任、技能等多维表现。
- 匿名评价列表只展示“匿名队友”，不暴露评价人。

### 3. 项目管理

进入“竞赛项目”页面。

重点讲解：

- 项目分为 `进行中`、`可评价`、`已归档`。
- 项目详情中可以添加队友。
- 只有进入“可评价”状态后，才适合进行队友互评。
- 项目详情展示评价进度，便于答辩说明业务闭环。

### 4. 提交匿名评价

进入“提交评价”页面，选择项目和队友。

重点讲解：

- 评价包含综合分和分项评分。
- 标签分为正向、中性、风险。
- 可填写文字评价和再次组队意愿。
- 后端限制同一项目同一评价对象只能评价一次。

### 5. 查看画像变化

提交评价后回到被评价人的画像页。

重点讲解：

- 评价会进入画像聚合。
- 雷达图、评分、标签和匿名评价列表都围绕评价数据生成。
- 隐藏状态的评价不参与前台公开展示。

### 6. 管理员审核后台

退出学生账号，使用管理员账号登录。

进入管理员后台。

重点讲解：

- 管理员可查看用户、项目、评价、申诉和统计信息。
- 对不适合公开展示的评价执行隐藏操作。
- 隐藏原因会保留在后台。
- 这是“匿名展示 + 后台追溯”的审核闭环。

### 7. 申诉闭环

切回学生账号，在自己的画像页对评价发起申诉。

再切回管理员账号处理申诉。

重点讲解：

- 学生不是直接删除评价，而是提交申诉理由。
- 管理员审核后处理，保留管理痕迹。
- 体现内容治理与平台边界意识。

## 答辩讲解重点

- 业务背景：竞赛组队后，队友协作表现难以沉淀，导致后续组队信息不透明。
- 核心方案：匿名评价 + 协作画像 + 管理员审核。
- 技术实现：Vue 3 前端、Express REST API、MySQL 数据建模、JWT 鉴权。
- 数据可视化：ECharts 雷达图让评分结果更直观。
- 安全边界：前台匿名，后台可追溯，管理员可审核。
- Demo 边界：不用于真实学校评价系统，不包含生产级风控。

## 常见问题

### 为什么叫匿名评价，但数据库还保存 reviewer_id？

匿名是前台展示策略，不是数据库彻底匿名。正式系统需要在匿名展示和争议追溯之间取得平衡。本 Demo 用后台保留真实关联来支持管理员审核和申诉处理。

### 为什么没有复杂推荐算法？

项目定位是课程设计 Demo，重点展示业务闭环和全栈实现。当前推荐和筛选以技能方向、标签、评分等基础数据为主，没有实现机器学习或复杂排序模型。

### 是否可以用于真实校园环境？

不建议直接使用。真实场景需要更严格的隐私合规、身份认证、内容审核、申诉流程、数据权限和风控机制。
